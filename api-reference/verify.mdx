## Endpoint

Verify that a blockchain payment transaction meets the expected criteria.

<ParamField path="transaction_hash" type="string" required>
  The blockchain transaction hash (must start with 0x, 66 characters) Example:
  `0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef`
</ParamField>

<ParamField path="chain" type="string" required>
  Blockchain network: `base` (Base Mainnet) or `base-sepolia` (Base Sepolia Testnet)
</ParamField>

<ParamField path="seller_address" type="string" required>
  The seller's blockchain address (42 characters, must start with 0x) Example:
  `0x742d35cc6634c0532925a3b844bc9e7595f0beb0`
</ParamField>

<ParamField path="expected_amount" type="string" required>
  Expected payment amount in wei (as string to avoid precision loss) Example:
  `"1000000000000000000"` (1 ETH in wei)
</ParamField>

<ParamField path="expected_token" type="string">
  Expected token contract address for ERC20 payments. Use `null` for native
  token (ETH, MATIC, etc.) Example: `0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48`
  (USDC)
</ParamField>

<ParamField path="metadata" type="object">
  Additional metadata for the transaction
  
  ```json
  {
    "order_id": "ORDER-123",
    "customer_id": "CUST-456"
  }
  ```
</ParamField>

<ParamField path="webhook_url" type="string">
  URL to receive settlement status updates. Must be HTTPS in production.
  Example: `https://your-app.com/webhooks/settlement`
</ParamField>

## Headers

<ParamField header="X-API-Key" type="string" required>
  Your API key for authentication
</ParamField>

<ParamField header="Idempotency-Key" type="string" recommended>
  Unique key to prevent duplicate verifications. Valid for 24 hours. Example:
  `verify_ORDER-123_1234567890`
</ParamField>

## Response Fields

<ResponseField name="verification_id" type="string" required>
  Unique identifier for this verification. Use this for settlement.
</ResponseField>

<ResponseField name="status" type="string" required>
  Verification status: `verified`, `pending`, `failed`, or `rejected`
</ResponseField>

<ResponseField name="transaction_hash" type="string" required>
  The verified transaction hash
</ResponseField>

<ResponseField name="chain" type="string" required>
  The blockchain network
</ResponseField>

<ResponseField name="verified_amount" type="string">
  Actual verified amount in wei (null if pending)
</ResponseField>

<ResponseField name="verified_token" type="string">
  Verified token contract address (null for native token)
</ResponseField>

<ResponseField name="verified_at" type="string">
  ISO 8601 timestamp when verification completed
</ResponseField>

<ResponseField name="details" type="object">
  Additional verification details (confirmations, block number, etc.)
</ResponseField>

  Verifications are processed in less than 2 seconds. Background workers handle
  transaction logging asynchronously.
</Info>

  request might be retried.
</Warning>
